# 午睡確認支援アプリ

|        |                  |
| ------ | ---------------- |
| Author | Tomokatsu Kumata |
| Date   | 2026/03/10       |
| Status | Updated          |

## 概要

午睡時の確認作業を支援するアプリです。iPhone / iPad のカメラを利用して、園児の姿勢を定量的に評価し、うつ伏せ寝のリスクを可視化します。

## 背景

現在、多くの保育施設で昼寝時に園児がうつ伏せになっていないか、職員が目視で確認しています。園児数が多い中での目視確認は負担が大きく、また見落としのリスクも存在します。高価な IoT デバイスの導入が難しい現場において、既存の端末（中古 iPhone 等）を活用した安価で高精度な補助ツールが求められています。

## 目的

- AIによる人体検出と顔の3D姿勢推定（FFI: Face Frontality Index）を組み合わせることで、うつ伏せ寝のリスクを定量的に評価し、事故の未然防止を支援すること。
- ファインダ内の全対象を同時に評価し、リスクスコアとして可視化することで、職員の目視確認を効率化すること。

## ユースケース

- 職員が部屋で iPhone / iPad をかざす。
- 複数の園児が昼寝している部屋（薄暗い環境を含む）で使用。
- 片手で操作可能なシンプルなインターフェース。

## 機能要件

- カメラ内の「人」を自動検出し、個別にリスクスコアを算出します。
- **リスク評価アルゴリズム**: 顔の正対度（Yaw/Pitch）と検出信頼度を統合した独自スコア $R$ を算出。
- **3段階の視覚的フィードバック**:
  - **緑色 (SAFE)**: リスクスコア低（$R < 0.15$）。仰向け状態。
  - **黄色 (CAUTION)**: リスクスコア中（$0.15 \le R < 0.40$）。顔の傾きや部分的な隠れ。
  - **赤色 (DANGER)**: リスクスコア高（$R \ge 0.40$）。うつ伏せ、または顔が完全に隠れている状態。
- **顔消失検知**: 人体は検出されているが顔が認識できない場合、うつ伏せとみなして最大リスク（赤色）を表示。
- アプリ起動時の注意事項同意フローと、次回以降のスキップ機能。
- 映像データは端末内で即時処理され、保存・送信は一切行わない。

## 技術選定

- **言語/フレームワーク**: Swift, SwiftUI
- **AIエンジン**: Apple Vision Framework (VNDetectFaceLandmarksRequestRevision3)
- **ハードウェア**: iPhone / iPad (iOS 15.0以上推奨)

## 制限・注意事項

- 本アプリは補助ツールであり、医療機器ではありません。最終的な確認は必ず人間が行う必要があります。
- 精度は照明環境や寝具（毛布等）による遮蔽状況に依存します。

## Pros

- 専用ハードウェア不要で導入コストが極めて低い。
- 3D姿勢推定により、従来の平面的な検知より高精度。

## Cons

- 身体が完全に毛布に隠れている場合は検知不能。
- 医療機器認証がないため、法的な保証はできない。
